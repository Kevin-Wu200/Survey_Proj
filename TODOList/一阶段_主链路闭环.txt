□ 1.1 ROS2 通信基础环境搭建
    - 安装 ROS2 Humble + Gazebo Garden
    - 配置 DDS 通信（Fast-DDS / CycloneDDS）
    - 验证 UAV 节点 ↔ 地面站节点 ↔ UGV 节点 三方互通
    - 确认 Topic/Service/Action 基本通信正常
    输出：ros2 topic list / ros2 node list 可看到三节点

□ 1.2 UAV 仿真节点最小原型
    - 在 Gazebo 中搭建 M300 RTK 四旋翼模型（含 GPS/IMU 传感器插件）
    - 实现起飞、悬停、降落三个基础动作
    - 通过 ROS2 Topic 发布无人机位姿（/uav/pose）
    - 模拟倾斜相机定时触发拍照（发布 /uav/image_raw）
    输出：Gazebo 中无人机可起飞悬停，地面站可收到位姿和图像 Topic

□ 1.3 UGV 仿真节点最小原型
    - 在 Gazebo 中搭建四轮差速底盘模型
    - 搭载 LiDAR（16线仿真）、双目相机、IMU 传感器插件
    - 实现键盘遥控移动（teleop_twist_keyboard）
    - 通过 ROS2 Topic 发布无人车位姿和 LiDAR 点云
    输出：Gazebo 中无人车可遥控移动，地面站可收到位姿和点云

□ 1.4 地面站数据处理管道（骨架）
    - 搭建 ROS2 数据接收节点，订阅 UAV/UGV 所有 Topic
    - 实现数据录制（ros2 bag record）基础功能
    - 地面站本地 SSD 缓存目录结构设计
    输出：可录制并回放 UAV/UGV 运行过程的数据包

□ 1.5 Web 端远程监测最小原型
    - 搭建 Vue3 + TypeScript + 高德地图 JS API v2.0 前端骨架
      （npm create vue@latest 脚手架初始化）
    - 后端搭建（Python FastAPI + WebSocket，uvicorn 直接启动）
    - 前端启动方式：npm run dev（Vite 开发服务器）
    - 实现：
      □ UAV/UGV 实时位置在高德地图上标绘（经纬度坐标）
      □ 无人机/无人车基本状态显示（连接状态、经纬度）
      □ 实时位姿数据通过 WebSocket 推送至前端
    输出：Web 页面可看到 UAV/UGV 图标在地图上移动

□ 1.6 主链路联调验证
    - 场景：无人机起飞执行简单航线 → 无人车同时遥控移动
    - 地面站接收全部数据并推送至 Web 前端
    - 验证端到端延迟（<500ms 为合格）
    输出：链路闭环演示成功，记录延迟指标
