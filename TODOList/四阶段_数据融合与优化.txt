====================================================================
四阶段 · 数据融合与性能优化
====================================================================
目标：实现在线三维重建数据融合管道，完成全链路性能调优。

预计工期：第6~10周（与三阶段 3B 并行）
依赖：三阶段协同 SLAM ✅（需空地位姿对齐作为融合输入）

--------------------------------------------------------------------
□ 4.1 数据融合管道搭建
    - 搭建 ROS2 Component 节点管道：
      □ image_rectify_node（图像校正）
      □ pointcloud_filter_node（点云滤波降采样）
      □ sfm_node（增量式 SfM 生成稀疏点云）
      □ icp_registration_node（ICP 精配准）
      □ meshing_node（三角网格生成）
    - 验证管道逐节点数据流正常
    输出：管道节点全部启动，Topic 数据流连通

□ 4.2 粗配准实现
    - 利用协同 SLAM 输出位姿 Ti_uav 和 Tj_ugv
    - 将 UAV SfM 点云与 UGV LiDAR 点云变换至同一坐标系
    - 验证初始对齐精度（RMSE < 20cm）
    输出：两组点云在同一坐标系下可视化

□ 4.3 精配准实现
    - 基于 ICP 算法进行 point-to-plane 精配准
    - 备选 NDT 算法进行比较
    - 目标函数：min Σ ||p_uav - T · q_ugv_nn||²
    - 精配准后 RMSE < 5cm
    输出：融合点云精度达标

□ 4.4 数据持久化系统
    - 搭建 PostgreSQL 15 + PostGIS 3 数据库（系统直接安装，非 Docker）
    - 提供数据库安装配置文档（Ubuntu 22.04 apt 安装步骤）
    - 设计测绘元数据表结构（任务 / 航点 / 轨迹 / 模型元数据）
    - 三维模型文件以文件系统存储，PostgreSQL 维护索引
    - 实现数据写入与查询 API
    输出：融合结果可存入数据库并通过 API 检索

□ 4.5 Web 前端功能增强（二）
    - 数据采集进度展示（已采集面积 / 预计完成时间）
    - 融合成果缩略图预览（待办：V2.0 优先级的简略实现）
    - 数据库查询接口对接（任务列表 / 历史成果查看）
    - 2D 地图世界配准工具集成（支持多场景 .pgw 自动生成）
    输出：Web 端可查看历史任务和成果概览

□ 4.6 全链路性能调优
    - 采集效率：单位面积采集耗时（目标 < 5 min/ha）
    - 融合延迟：从数据采集到成果输出（目标 < 10 min）
    - 通信带宽优化（按需发布 / QoS 配置调优）
    - 前端渲染性能优化（点云抽稀显示 / LOD 策略）
    输出：全链路时延指标达标
