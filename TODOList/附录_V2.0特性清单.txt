====================================================================
附录 · V2.0 特性清单
====================================================================
说明：以下功能在当前版本（V1.0）中优先级较低，视工期情况安排在
V2.0 迭代中实现。

--------------------------------------------------------------------
□ V2.1 纹理映射
    - 将无人机高分辨率倾斜影像映射至融合后三角网格表面
    - 生成带纹理的真实感三维模型
    - 涉及：纹理展开（UV Unwrapping）、纹理融合（色彩校正/
      接缝消除）、LOD 纹理层级生成
    优先级：中
    依赖：四阶段数据融合管道 ✅

□ V2.2 实时三维重建预览
    - 在 Web 前端使用 Three.js 实时渲染融合中的三维模型
    - 支持增量式 Mesh 更新（边重建边显示）
    - 支持视角旋转 / 缩放 / 剖切
    - 前端技术栈：Vue3 + TypeScript + Three.js（与 V1.0 统一，Three.js 已为地图容器引擎）
      ✓ V1.0 已实现：Three.js 3D 场景 + GLB 模型加载 + OrbitControls + 2D 地图纹理叠加
    优先级：低
    依赖：V2.1 纹理映射 ✅ / 四阶段数据融合管道 ✅

□ V2.3 数据采集进度与成果预览
    - 已采集面积 / 预计完成时间实时估算
    - 采集覆盖率热力图（哪些区域已采集、哪些有空洞）
    - 成果缩略图自动生成与网格展示
    优先级：中
    依赖：一阶段主链路闭环 ✅

□ V2.4 协同路径规划优化
    - 空地同步采集过程中的动态视点协调
    - UAV 自动调整航线以覆盖 UGV 即将到达的区域
    - UGV 自动调整路径以补拍 UAV 盲区
    优先级：低
    依赖：三阶段协同 SLAM ✅

□ V2.5 移动端轻量指挥界面
    - 基于 Vue3 + TypeScript 的 PWA 实现（与 V1.0 前端技术栈统一）
    - 现场人员通过手机/pad 查看任务状态和告警
    - 简化版航线下发功能
    优先级：低
    依赖：二阶段 Web 前端 ✅

====================================================================
生命周期管理
====================================================================
以下是对 V1.0 阶段已有功能在 V2.0 中的演进规划：

□ 协同 SLAM → 支持多 UAV + 多 UGV 编队协同
□ 数据融合 → 支持语义标签自动添加（建筑物/植被/道路分类）
□ 仿真场景 → 增加水下/隧道等特殊测绘场景
□ 系统安全 → 引入链路加密（TLS）与重放攻击防护
□ 数据管理 → 引入数据版本管理（增量版本 / 回滚）
□ CI/CD → 集成仿真测试自动化（Gazebo 自动运行 + 指标评估）
