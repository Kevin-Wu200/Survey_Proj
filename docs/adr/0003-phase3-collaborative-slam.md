# ADR-0003: 协同 SLAM 分层架构设计

## 状态

已接受 (2026-05-14)

## 背景

空地协同无人化智能测绘系统需要实现 UAV（无人机）与 UGV（无人车）的协同 SLAM（同步定位与建图）。核心思路是"UAV 全局拓扑约束 + UGV 局部精细建图"的分层架构：UAV 高空俯瞰提供全局拓扑约束和回环检测辅助，UGV 地面行驶完成精细局部建图。

## 决策

### 1. 分层架构

```
┌──────────────────────────────────────────────┐
│              UAV 全局拓扑建图 (3A.1)           │
│   RTK 位姿 + 增量 SfM → 拓扑图 G=(V,E)       │
│   关键帧节点 + 共视边 + 描述子 di ∈ R¹²⁸      │
└──────────────────┬───────────────────────────┘
                   │  T_uav2ugv
┌──────────────────▼───────────────────────────┐
│          空地位姿协同优化 (3A.3)               │
│   手眼标定 + 滑动窗口联合优化                  │
└──────────────────┬───────────────────────────┘
                   │
┌──────────────────▼───────────────────────────┐
│          UGV 多传感器融合定位 (3A.2)           │
│   FAST-LIO2 LiDAR-IMU + 双目视觉 + 因子图     │
└──────────────────┬───────────────────────────┘
                   │
┌──────────────────▼───────────────────────────┐
│          UAV 辅助回环检测 (3A.4)               │
│   拓扑图搜索 + 跨视角匹配 + PGO 全局优化       │
└──────────────────────────────────────────────┘
```

### 2. 模块划分

| 模块 | 文件 | 功能 |
|------|------|------|
| UAV 拓扑建图 | `uav_topology.py` | RTK位姿+增量SfM，拓扑图构建 |
| UGV 融合定位 | `ugv_fusion.py` | LiDAR-IMU紧耦合+双目视觉+因子图 |
| 协同优化 | `collaborative_optimizer.py` | 手眼标定+滑动窗口联合优化 |
| 回环检测 | `loop_closure.py` | 跨视角匹配+SuperPoint/SuperGlue+PGO |
| 实时性优化 | `realtime_opt.py` | 异步因子图+Scan-to-Map加速+ORB降级 |
| 鲁棒性增强 | `robustness.py` | 传感器健康监控+退化检测+模式切换 |
| 一致性校验 | `consistency.py` | 重叠检测+漂移检测+自动重校正 |

### 3. 数据流

1. UAV 飞行时通过 RTK 获取精确位姿，结合相机图像进行增量式 SfM
2. 构建拓扑图 G=(V,E)，节点存储位姿 Ti 和描述子 di
3. UGV 行驶时通过 LiDAR-IMU-视觉多传感器融合实时定位
4. 通过手眼标定建立 UAV↔UGV 坐标系变换
5. 在 UAV 拓扑图中搜索 UGV 邻近关键帧作为回环候选
6. 跨视角特征匹配验证回环，加入因子图进行 PGO 全局优化
7. 定期校验两张地图的一致性，检测漂移并触发重校正

### 4. 容错策略

- 视觉失效 → 降级为纯 LiDAR-IMU 定位
- GNSS 丢失 → 纯视觉/激光定位保持
- 回环误匹配 → 几何一致性校验 + RANSAC 剔除
- 推理帧率不足 → ORB 降级或降低检测频率

## 后果

- **优点**: 分层架构解耦 UAV 全局约束与 UGV 局部建图，易于独立开发和测试
- **缺点**: 需要 UAV 和 UGV 时间同步良好；手眼标定精度直接影响协同效果
- **风险**: 大规模场景下拓扑图可能过大；后续需考虑图稀疏化和增量更新策略

## 实现细节
